# DELTA Protocol v2.7.3 — Documentation Refresh

Status: Documentation-only milestone  
Scope: Export, Intent Create, Portable Bundle, and public reviewer workflow

## 1. Purpose

v2.7.3 refreshes DELTA documentation after the UX/adoption milestones:

- v2.7.0 — DELTA Export
- v2.7.1 — Intent Create Helper
- v2.7.2 — Portable DELTA Bundle

This release does not add new proof functionality.

It aligns the reviewer-facing documentation with the current proof sharing workflow:

```text
delta-record.json
→ unsigned intent draft
→ exported Markdown/HTML report
→ portable .delta bundle
→ external review
```

## 2. Current UX / adoption flow

A user can now:

1. Verify the existing DELTA proof baseline:

```powershell
python src/delta_cli.py verify-all
```

2. Verify canonical JSON vectors:

```powershell
python tools\delta_canonical_json.py verify-vectors --vectors tests\vectors\canonical-json\vectors.json
```

3. Create an unsigned intent draft bound to a full DELTA record hash:

```powershell
python tools\delta_intent_create.py create --record path\to\delta-record.json --issue ISSUE-ID --purpose "Purpose" --created-by local-operator --role operator --policy-id intent-policy-v1 --enforcement report_only --out intent-attestation.json
```

4. Export a technical report:

```powershell
python tools\delta_export.py export --record path\to\delta-record.json --format html --out delta-report.html
```

5. Package public artifacts into a portable `.delta` bundle:

```powershell
python tools\delta_bundle.py create --output sample.delta --record path\to\delta-record.json --intent intent-attestation.json --report delta-report.html
```

6. Verify the bundle:

```powershell
python tools\delta_bundle.py verify --bundle sample.delta --dir extracted
```

## 3. Security boundaries preserved

v2.7.3 reinforces that DELTA reports and bundles:

- do not create new cryptographic proofs,
- do not replace proof-specific verifiers,
- do not prove legal truth,
- do not prove real-world truth,
- do not prove human identity,
- do not prove wallet balance,
- do not prove regulatory compliance,
- must not contain private keys, secrets, raw private evidence, or decrypted evidence.

## 4. Relationship to Signed Bundle

Signed Bundle work is intentionally deferred until after this documentation refresh.

Rationale:

- v2.7.0, v2.7.1, and v2.7.2 changed the user-facing proof-sharing workflow.
- Reviewers need a current map before new signature functionality is added.
- v2.8.0 Signed Bundle should build on a documented v2.7.x UX baseline.

## 5. Next planned milestone

After this documentation-only refresh, the next security hardening milestone is:

```text
v2.8.0 — Signed DELTA Bundle
```

Signed Bundle should add detached Ed25519 signatures over the exact `.delta` bundle hash, without replacing `delta_bundle.py verify` or proof-specific DELTA verifiers.
