Type: Documentation-only audit clarification milestone

## Added / Changed

- updated `verifier/ts/README.md`

## Purpose

This release adds audit-driven clarification for three reviewer-sensitive areas:

- canonical JSON cross-language determinism,
- Proof of Replay nondeterminism and environment assumptions,
- Bitcoin BIP-322 external / shape-only boundary.

## Scope

Documentation only.

No new cryptographic functionality.

No verifier behavior changes.

## Security

This clarification reduces overclaiming risk.

It explicitly states that DELTA:

- uses a strict canonical JSON profile with frozen vectors,
- treats replay determinism as a declared-assumption boundary,
- keeps Bitcoin BIP-322 external profile conservative unless full local cryptographic verification exists,
- must not claim legal identity, signer authority, real-world truth, regulatory compliance, or correctness of input data before signing.
