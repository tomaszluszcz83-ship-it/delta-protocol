# DELTA v2.11.0 — TypeScript CLI Contract Tests Changelog Entry

Type: Integration / contract test milestone

## Added

- `tools/delta_ts_cli_contract_tests.py`
- `docs/typescript-cli-contract-tests-v2.11.0.md`

## Purpose

This release adds Python-driven integration tests for the TypeScript CLI JSON output contract introduced in v2.10.2.

It validates stable machine-readable behavior for selected TypeScript verifier commands.

## Scope

Tested behavior includes:

- TypeScript build,
- TypeScript canonical JSON vectors,
- TypeScript schema verification,
- signed record demo creation,
- JSON output success path,
- JSON output usage error path,
- JSON output verification failure path,
- top-level JSON fields,
- exit code categories.

## Exit code categories

| Exit code | Code name |
| --- | --- |
| 0 | `OK` |
| 1 | `VERIFICATION_FAILED` |
| 2 | `USAGE_ERROR` |
| 3 | `INTERNAL_ERROR` |

## Security

This release adds no new cryptographic functionality.

It does not claim full Python ↔ TypeScript feature parity.

It tests the TypeScript CLI machine-readable output contract only.

## Tested

Expected checks:

```powershell
python tools\delta_ts_cli_contract_tests.py

python src/delta_cli.py verify-all
python tools\delta_canonical_json.py verify-vectors --vectors tests\vectors\canonical-json\vectors.json
git diff --check
```
