# DELTA v2.16.7 — GitHub Actions Demo Workflow / Automated Tamper Detection Check

## Executive summary

DELTA v2.16.7 introduces a minimal GitHub Actions workflow for the public tamper-detection demonstration.

The workflow is intentionally reduced to the smallest stable CI-visible demonstration of the foundational Proof of Change intuition:

```text
known artifact bytes → verification OK → tamper → hash mismatch → verification FAIL
```

This release does not add new cryptographic functionality and does not modify DELTA verification semantics. It provides a public CI demonstration that can be shown to developers, reviewers, auditors, and early adopters.

## What changed

Added or updated:

## Stabilization decision

The workflow uses `ubuntu-latest`, `bash`, and `sha256sum`.

This avoids platform-specific ambiguity from PowerShell, Windows runner behavior, checkout line endings, script encoding, CRLF/LF conversion, and Python dependency installation. The purpose of this workflow is not to exercise the full DELTA reference implementation. The purpose is to make the public tamper-detection demonstration reliable, visible, and understandable.

Full DELTA verification remains covered by the existing DELTA verification checks and reference tools.

## Security impact

CI workflow and documentation only.

No cryptographic verification semantics are changed.

No proof profile is modified.

No verifier behavior is changed.

No implementation behavior is changed.

No schemas, signatures, canonicalization rules, bundle rules, hash-binding rules, or proof verification logic are modified.

This release reinforces DELTA’s anti-overclaiming discipline by keeping the CI demo explicitly scoped to minimal hash-based tamper detection.

## Tests expected

The GitHub Actions workflow is expected to show:

```text
OK: Original artifact hash matches expected value.
OK: Tampered artifact hash mismatch detected.
OK: Demo succeeded: tampering was detected.
```

The workflow should pass only when tampering is detected.
