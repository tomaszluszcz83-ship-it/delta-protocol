# DELTA v2.8.1 — Revocation / Invalidation Design Changelog Entry

Type: Documentation-only design release

## Added

- Revocation and invalidation design.
- Key compromise playbook.
- Revocation trust ledger profile.

## Purpose

This release defines how DELTA should handle:

- compromised keys,
- invalidated artifacts,
- superseded artifacts,
- revoked bundle signatures,
- revoked trust delegations,
- withdrawn intent attestations,
- policy changes.

## Security

- Revocation is represented as a new event, not a mutation or deletion of old artifacts.
- Cryptographic validity is separated from trust validity.
- Signed artifacts can remain cryptographically valid while becoming untrusted under policy.
- Revocation does not prove legal truth, real-world truth, legal identity, signer authority, or regulatory compliance.
