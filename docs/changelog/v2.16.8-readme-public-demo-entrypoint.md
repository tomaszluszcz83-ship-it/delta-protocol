# DELTA v2.16.8 — README Public Demo Badge / Reviewer Entry Point

## Executive summary

DELTA v2.16.8 introduces a README-level public demo and reviewer entry point.

After v2.16.6 added the minimal public tamper-detection demo and v2.16.7 automated that demonstration in GitHub Actions, this milestone makes those public adoption assets visible directly from the repository front page.

The purpose is to reduce first-contact friction for external developers, auditors, protocol reviewers, security researchers, software supply-chain engineers, CI/CD maintainers, potential integrators, and serious early adopters.

## Strategic purpose

DELTA now has a public adoption path, but that path must be immediately visible to a new visitor.

This milestone places the most important public entry points near the top of the README:

- minimal local demo,
- automated GitHub Actions tamper-detection workflow,
- quick-start documentation,
- reviewer checklist,
- external review request.

The goal is to move a first-time visitor from passive reading into direct verification, structured review, and responsible feedback.

## Security impact

Documentation only.

No cryptographic verification semantics are changed.

No proof profile is modified.

No verifier behavior is changed.

No implementation behavior is changed.

No schemas, signatures, canonicalization rules, bundle rules, hash-binding rules, or proof verification logic are modified.

The README entry point explicitly preserves DELTA’s anti-overclaiming boundary by clarifying that the minimal public demo demonstrates hash-based tamper detection only and is not full signed `.delta` bundle verification.
